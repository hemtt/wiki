# bin-website - Metadata Generator for HEMTT Wiki

A Rust-based metadata generator that produces JSON files consumed by the Angular frontend website.

## Features

- **Pure Rust**: Lightweight metadata generation
- **JSON Output**: Machine-readable format for Angular frontend
- **Fast**: Generates metadata in under 1 second
- **Separated Concerns**: Backend generates data, frontend renders UI
- **GitHub Pages Ready**: Integrates with Angular build pipeline

## Quick Start

```bash
# Build the metadata generator
cargo build -p arma3-wiki-website --release

# Generate metadata JSON files
./target/release/website

# Output created in: dist-website/assets/data/
# - commands.json
# - filters.json
```

## Purpose

This generator transforms command YAML files and test reports into structured JSON metadata that the Angular frontend loads at runtime for display and filtering.

## Architecture

```
Input                    Processing          Output
─────────────────────────────────────────────────────
dist/commands/*.yml  ──┐
                      ├─→  Rust Parser  ──→  dist-website/assets/data/
dist/report.json ─────┘                       ├── commands.json
                                              └── filters.json
```

The Angular application loads these JSON files and handles all rendering and filtering client-side.

## Project Structure

```
bin-website/
├── Cargo.toml                 # Workspace member definition
└── src/
    ├── main.rs                # Entry point
    │   - Orchestrates metadata generation
    │   - Handles file I/O
    │   - Creates output directories
    ├── models.rs              # Data structures
    │   - Command struct (name, description, status, groups, example, parameters)
    │   - CommandStatus enum (Passed, Failed, Outdated, Unknown)
    │   - Report struct (maps command names to statuses)
    │   - load_commands() and load_report() functions
    └── generator.rs           # Generation logic
        - generate_commands_file() - Creates commands.json
        - generate_filters_file() - Creates filters.json
```

## Input Data

### Command Files

Location: `dist/commands/*.yml`

Each file contains a single command definition:
```yaml
name: actionId
description: Gets an action ID for the specified action
syntax: actionId action
parameters:
  - name: action
    description: The action name
example: actionId playable player
```

Generated by `bin-parse`. Run to regenerate:
```bash
cargo run -p arma3-wiki-parse -- <path-to-commands>
```

### Status Report

Location: `dist/report.json`

Contains test execution results:
```json
{
  "commands": {
    "actionId": "Passed",
    "activatedAddons": "Failed",
    "addAction": "Outdated"
  }
}
```

Generated by `bin-parse`. Shows which commands have been tested and their status.

## Output Format

### commands.json

Complete command catalog with all details:

```json
{
  "commands": [
    {
      "name": "actionId",
      "description": "Gets an action ID for the specified action",
      "status": "Passed",
      "groups": ["Available", "Action IDs"],
      "example": "actionId playable player",
      "parameters": [
        {
          "name": "action",
          "description": "The action name"
        }
      ]
    },
    { /* ... more commands ... */ }
  ]
}
```

### filters.json

Available filter options for the UI:

```json
{
  "statuses": ["Passed", "Failed", "Outdated", "Unknown"],
  "groups": ["Available", "Action IDs", "Inventory", "Strings", ...],
  "sortOptions": [
    { "label": "Name (A-Z)", "value": "name_asc" },
    { "label": "Name (Z-A)", "value": "name_desc" },
    { "label": "Status", "value": "status" }
  ]
}
```

## Integration with Angular

The `CommandService` in the Angular app loads these files:

```typescript
// CommandService
loadCommands(): Observable<Command[]> {
  return this.http
    .get<{ commands: Command[] }>('assets/data/commands.json')
    .pipe(map(data => data.commands));
}

loadFilters(): Observable<FilterOptions> {
  return this.http.get<FilterOptions>('assets/data/filters.json');
}
```

## Customization

### Add Custom Fields to Commands

1. **models.rs** - Add field to `Command` struct:
   ```rust
   pub struct Command {
       pub name: String,
       pub description: String,
       pub custom_field: String,  // ← Add here
   }
   ```

2. **generator.rs** - Include in JSON output:
   ```rust
   json!({
       "name": cmd.name,
       "description": cmd.description,
       "custom_field": cmd.custom_field,  // ← Add here
   })
   ```

3. **Angular** (`website/src/app/services/command.service.ts`) - Update interface:
   ```typescript
   interface Command {
     name: string;
     description: string;
     customField: string;  // ← Add here
   }
   ```

### Modify Filter Options

Edit `generator.rs` `generate_filters_file()`:

```rust
fn generate_filters_file() {
    // Add new filter categories
    // Extract unique values from commands
    // Define sort options
}
```

### Change Output Location

Edit `main.rs` to change where files are written:

```rust
let output_dir = PathBuf::from("custom/output/location");
```

## Troubleshooting

### "No such file or directory" error

```
Error: failed to read directory 'dist/commands'
```

**Solution**: Ensure commands exist:
```bash
cargo run -p arma3-wiki-parse
cargo run -p arma3-wiki-replace  # If needed
# Then retry bin-website
```

### "expected value at line 1 column 1" (JSON error)

Malformed YAML in command files.

**Solution**: Validate YAML:
```bash
cat dist/commands/actionId.yml  # Check for syntax errors
```

### Empty commands.json

Commands not loading from YAML files.

**Solution**: Check file paths in `main.rs`:
```rust
let commands_dir = PathBuf::from("dist/commands");
// Verify this path exists and has *.yml files
```

### Angular shows empty page

Frontend not loading JSON.

**Solution**: Check:
1. Files exist: `ls dist-website/assets/data/commands.json`
2. Browser Network tab: See if commands.json is 404
3. Browser Console: Any fetch errors?
4. HTTP server is configured correctly
